# Brainstorm Workflow

## Purpose
Facilitate structured ideation and problem exploration to generate innovative solutions and uncover hidden requirements.

## Trigger
- New project initiation
- Feature exploration request
- Problem-solving session needed
- Innovation sprint

## Agents Involved
| Agent | Role | Model |
|-------|------|-------|
| business-analyst | Facilitator, requirements gathering | sonnet |
| search-specialist | Research support, competitive intel | haiku |
| architect-review | Technical feasibility assessment | opus |

## Workflow Steps

### Step 1: Problem Definition
**Agent:** business-analyst
**Actions:**
1. Clarify the problem statement with stakeholder
2. Identify constraints and boundaries
3. Define success criteria
4. Map affected user personas

**Output:** Problem statement document

### Step 2: Research Phase
**Agent:** search-specialist
**Actions:**
1. Research existing solutions
2. Identify industry best practices
3. Gather competitive intelligence
4. Find relevant case studies

**Output:** Research summary

### Step 3: Ideation
**Agent:** business-analyst
**Actions:**
1. Generate solution options (minimum 3)
2. Map each option to constraints
3. Identify pros/cons for each
4. Rank by feasibility and impact

**Output:** Solution options matrix

### Step 4: Technical Validation
**Agent:** architect-review
**Actions:**
1. Assess technical feasibility of top options
2. Identify architectural implications
3. Estimate complexity and effort
4. Flag potential risks

**Output:** Technical feasibility report

### Step 5: Synthesis
**Agent:** business-analyst
**Actions:**
1. Combine all inputs
2. Create recommendation
3. Define next steps
4. Prepare stakeholder presentation

**Output:** Brainstorm summary with recommendations

## Exit Criteria
- Problem clearly defined
- At least 3 solution options explored
- Technical feasibility confirmed for recommended option
- Next steps agreed
